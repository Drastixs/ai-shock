AUVON AS8016 — Control API Reference

A REST abstraction over the AUVON AS8016 4-channel TENS / EMS muscle stimulator. Every endpoint maps a physical button press or LCD readout to a CRUD operation.

The physical device is driven by buttons (Mode M, Timer T, Channel-select, +, -) plus four output relays. This API exposes Channel, Max output, Mode, Outputs (relay routing), and Device (timer, E-stop, status). GET returns remembered state; PUT actuates buttons until the target is reached; POST fires a one-shot action.

This contract is implemented on the Axiometa Genesis Mini (MicroPython) — served by lib/http_api.py over lib/tens_controller.py. The device driver (5 button-bypass MOSFETs + 4 output relays) sits underneath.
State is a software mirror. The AS8016 gives no digital feedback, so every GET returns the state the controller has commanded (remembered per channel), not a reading of the LCD. If the unit is also operated by hand, re-issue a PUT to a known value to resync.

Base URL & conventions

Base URL
http://<device-ip>:8080/api/v1
Content-Type
application/json
Read ops
GET — reflects the current LCD state
Write ops
PUT (set to value) / POST (fire a button action)
Success
200 OK for reads and settled writes
Failure
400 invalid value · 409 device busy/locked · 503 device offline

The device has two independent state groups (Channel A and Channel B). Select the target group with PUT /channel first — then /intensity and /mode read and write whichever channel is active, with no channel field of their own. There is no CREATE/DELETE for device state — the resource set is fixed by hardware — so the CRUD surface is Read + Update.

Channel A drives physical outputs A1A2 and Channel B drives B1B2. A and B can hold different modes and intensities simultaneously.

Channel

The active channel is the group currently flashing on the LCD — the one that +/- and M will affect. Mapped to the center channel-select button.

GET /channel Read which channel group is currently selected (flashing) on the display.
Request 
GET /api/v1/channel
200 OK 
{
  "active": "A",
  "available": ["A", "B"],
  "outputs": {
    "A": ["A1", "A2"],
    "B": ["B1", "B2"]
  }
}
PUT /channel Select the active channel group. Presses the channel-select button until the target group flashes.
FieldTypeRequiredDescription
activestringrequiredTarget group — "A" or "B".
Request 
PUT /api/v1/channel
{ "active": "B" }
200 OK 
{
  "active": "B",
  "changed": true
}

Max output (intensity)

Output strength for the active channel. 20 discrete levels, stepped with the +/- buttons. Select the target group with PUT /channel first; intensity then applies to whichever channel is active.

Safety. Selecting a new mode resets intensity to the lowest level (device behaviour). Always ramp up from a low level; the device caps at level 20.
GET /intensity Read the current output level of the active channel.
Request 
GET /api/v1/intensity
200 OK 
{
  "level": 8,
  "max_level": 20
}
PUT /intensity Set the output level of the active channel. Fires +/- the required number of steps to reach the target.
FieldTypeRequiredDescription
levelintegerrequiredTarget level, 0–20. 0 = no output.
Request 
PUT /api/v1/intensity
{ "level": 12 }
200 OK 
{
  "level": 12,
  "steps_applied": 4
}
Values outside 0–20 return 400.

Mode

The stimulation program. 24 sub-modes across two master modes: TENS · P01–P16 for pain relief and EMS · P17–P24 for muscle performance. Cycled with the M button per active channel.

GET /modes List all available modes with their master mode and sensation description. Static catalogue.
200 OK (truncated) 
{
  "count": 24,
  "modes": [
    { "mode": "P01", "master_mode": "TENS", "feel": "Continuous comfortable tingling." },
    { "mode": "P03", "master_mode": "TENS", "feel": "Comfortable rhythmic tingling." },
    // ... P02, P04–P16 ...
    { "mode": "P20", "master_mode": "EMS",  "feel": "Muscle twitches at a very low frequency, like a tapping massage." },
    { "mode": "P23", "master_mode": "EMS",  "feel": "Fast-twitch pulse frequency; builds maximum muscle strength." }
  ]
}
GET /mode Read the mode currently running on the active channel.
Request 
GET /api/v1/mode
200 OK 
{
  "mode": "P03",
  "master_mode": "TENS",
  "feel": "Comfortable rhythmic tingling."
}
PUT /mode Set the mode on the active channel. Presses M until the target program is shown.
FieldTypeRequiredDescription
modestringrequiredTarget program, "P01"–"P24".
Request 
PUT /api/v1/mode
{ "mode": "P18" }
200 OK 
{
  "mode": "P18",
  "master_mode": "EMS",
  "intensity_reset": true
}
intensity_reset: true signals the device dropped this channel's level to 0 on mode change — re-set intensity via PUT /intensity afterward.

Outputs (relay routing)

Independently enable/disable each of the four physical output jacks — A1A2 (channel A) and B1B2 (channel B) — via the four output relays. This is separate from channel selection: selecting a channel chooses what +/-/M affect; enabling an output physically connects that jack. Boot default and /all_off leave every output disabled.

GET /outputs Read the enabled/disabled state of all four output jacks.
200 OK 
{ "A1": false, "A2": false, "B1": true, "B2": true }
PUT /outputs Enable/disable one jack, or both jacks of a channel at once.
FieldTypeRequiredDescription
jackstringone of jack / channel"A1", "A2", "B1", "B2".
channelstringone of jack / channel"A" or "B" — sets both jacks of the channel.
enabledboolrequiredtrue connects, false disconnects.
Request 
PUT /api/v1/outputs
{ "jack": "A1", "enabled": true }
200 OK 
{ "action": "output_A1:on" }

Device (timer, safety, status)

POST /timer Tap the Time button once (steps the session timer, 10–90 min on the device).
200 OK 
{ "action": "timer_adjust" }
POST /all_off E-stop. Releases all buttons and opens all output relays immediately.
Safety. /all_off opens every relay, so no output flows regardless of the mirrored intensity level. It does not change the device's remembered mode/level. This is also the boot default.
200 OK 
{ "action": "all_off" }
GET /status Full mirrored state snapshot: both channels' level+mode, output relays, uptime, Wi-Fi.
200 OK 
{
  "active_channel": "A",
  "channels": {
    "A": { "level": 0, "mode": "P01", "master_mode": "TENS" },
    "B": { "level": 0, "mode": "P01", "master_mode": "TENS" }
  },
  "outputs": { "A1": "disabled", "A2": "disabled", "B1": "disabled", "B2": "disabled" },
  "last_action": "all_off", "uptime_ms": 12345
}
Legacy flat aliases remain for convenience: POST /mode/next, POST /channel?ch=A, POST /intensity/up?steps=N, POST /output?jack=A1&enabled=true, POST /all_off, GET /status.
